## 二、详细升级方案

### 1. 需求澄清阶段 - 思维链与提示词优化

**目标模块**: `backend/app/agents/clarify/nodes.py`

#### 1.1 问题描述
当前需求澄清对话体验较为机械，引导问答缺乏自然的思考过程展示，用户难以感知 AI 的推理逻辑。

#### 1.2 优化方案

##### A. 引入 Chain-of-Thought (CoT) 思维链
在意图分析节点 (`intent_analyzer_node`) 中增加显式推理过程：

```python
# 优化前（当前）
INTENT_ANALYZER_PROMPT = """
请分析用户的最新回复，判断其意图并提取信息。
# 输出格式 (JSON)
{"intent": "...", "extracted_data": {...}}
"""

# 优化后（V1.1）
INTENT_ANALYZER_PROMPT = """
请按以下步骤分析用户回复：

**Step 1 - 理解用户意图**
分析用户说的内容，判断其核心目的。

**Step 2 - 关联上下文**
回顾已收集的需求信息，判断用户是否：
- 补充新信息
- 修改已有信息
- 请求帮助
- 确认开始创作

**Step 3 - 提取结构化数据**
从用户回复中提取与当前字段相关的具体信息。

**Step 4 - 生成响应**
基于以上分析，确定下一步行动。

# 输出格式 (JSON)
{
    "reasoning": "逐步思考过程（2-3句话）",
    "intent": "意图类型",
    "extracted_data": {...},
    "confidence": 0.0-1.0
}
"""
```

##### B. 优化引导生成节点的自然度
在 `guidance_generator_node` 中：

- 增加对已确认信息的**主动关联**能力
- 问题生成时参考完整的对话上下文，避免机械式提问
- 根据用户回答风格动态调整提问方式

##### C. 实现方案
- **文件**: `backend/app/agents/clarify/nodes.py`
- **修改点**:
  - `INTENT_ANALYZER_PROMPT` 常量（约第317行）
  - `GUIDANCE_GENERATOR_PROMPT` 常量（约第436行）
  - `intent_analyzer_node()` 函数返回值处理
- **新增**: `reasoning` 字段返回给前端（可选展示）

#### 1.3 验收标准
- [ ] 意图分析准确率提升至 90%+
- [ ] 引导问题与上下文关联度提升
- [ ] 用户单轮对话即可完成需求确认的比例提升

### 2. 需求确认书阶段 - 记忆管理与信息完整性

**目标模块**: `backend/app/agents/clarify/nodes.py`

#### 2.1 问题描述
需求确认书生成时存在**信息丢失**问题，部分在需求澄清阶段确认的设定未被正确保留到最终确认书中，导致后续剧本生成与用户原始意图偏差较大。

#### 2.2 问题根因分析
1. **摘要截断问题**: `format_messages_for_prompt()` 函数限制最大消息数为 50，长对话可能丢失早期信息
2. **JSON解析容错过度**: 当 LLM 返回非标准 JSON 时，回退方案使用默认值覆盖用户设定
3. **状态同步问题**: `requirements` 字典更新不完整，部分字段未被正确写入

#### 2.3 优化方案

##### A. 强化对话记忆管理
```python
# 优化方案：分层记忆策略
async def generate_requirement_confirmation(messages, llm_service):
    """
    V1.1 优化版本 - 分层记忆
    """
    # 1. 核心需求层：从 requirements 字典直接获取（最可靠）
    core_requirements = state.get("requirements", {})

    # 2. 关键对话层：提取包含需求关键词的对话片段
    key_messages = extract_key_requirement_messages(messages)

    # 3. 摘要层：生成对话摘要作为补充
    summary = await generate_conversation_summary(messages, llm_service)

    # 合并三层信息，优先级：核心需求 > 关键对话 > 摘要
    combined_context = merge_context_layers(core_requirements, key_messages, summary)
```

##### B. 增加需求字段完整性校验
```python
def validate_requirements_completeness(requirements: dict) -> tuple[bool, list]:
    """
    校验需求字段完整性

    Returns:
        (是否完整, 缺失字段列表)
    """
    required_fields = {
        "genre": "题材类型",
        "protagonist": "主角设定",
        "conflict": "核心冲突",
        "target_audience": "目标受众",
        "episodes": "集数",
        "style": "风格基调"
    }

    missing = []
    for field, desc in required_fields.items():
        if field not in requirements or not requirements[field]:
            missing.append(desc)

    return len(missing) == 0, missing
```

async def script_writer_node(state: ScriptState) -> ScriptState:
    # ... 生成剧本 ...

    max_retries = 2
    for attempt in range(max_retries):
        content = await llm_service.generate(prompt, max_tokens=2000)
        word_count = len(content)

        if 600 <= word_count <= 800:

        if word_count < 600:
