# 剧作大师 V1.1.5 升级文档

**版本号**: V1.1.5
**更新日期**: 2026-03-04
**更新类型**: 功能优化与体验提升

---

## 一、版本概述

V1.1.5 版本聚焦于**上下文管理优化**和**内容质量提升**，通过增强不同阶段间的信息传递、丰富人物小传维度、优化界面交互体验，全面提升剧本创作的整体质量和用户操作便利性。

---

## 二、主要升级内容

### 1. 需求澄清阶段上下文管理优化

#### 1.1 三层上下文架构

引入统一的三层上下文管理机制，解决长对话场景下的信息丢失问题：

| 层级 | 字段名 | 作用 | 更新时机 |
|------|--------|------|----------|
| **对话摘要** | `conversation_summary` | 记录完整对话历史（200字内） | 每5轮对话或关键节点 |
| **需求分析** | `requirement_analysis` | 当前对需求的理解（300字内） | 每轮对话后实时更新 |
| **消息索引** | `last_context_update_index` | 追踪上次更新位置 | 上下文更新时 |

#### 1.2 需求评估结构化

新增结构化需求评估体系，提升需求收集的精准度：

```typescript
interface RequirementAssessment {
  genre?: {
    status: 'empty' | 'partial' | 'confirmed'
    understanding: string      // 智能体对题材的理解
    confidence: number         // 置信度 0-1
    suggestion?: string        // 改进建议
  }
  protagonist?: { /* 同上 */ }
  conflict?: { /* 同上 */ }
  // ... 其他字段
}
```

#### 1.3 需求理解可视化

新增右侧边栏实时展示当前需求理解状态：

- **暂定剧名**: 根据当前信息智能生成
- **题材概述**: 对题材的精炼总结
- **主角概述**: 主角设定摘要
- **冲突概述**: 核心冲突描述
- **待完善项**: 动态显示下一步需确认的内容

### 2. 不同阶段间上下文传递优化

#### 2.1 需求确认书结构化增强

需求确认阶段生成结构化的需求确认书，确保信息完整传递给创作阶段：

```typescript
interface RequirementConfirmation {
  // 基础信息
  title: string                    // 剧名
  genre: string                    // 题材
  episodes: string                 // 集数
  target_audience: string          // 目标受众

  // 摘要字段（用于展示）
  protagonist_summary: string      // 主角设定摘要
  core_conflict_summary: string    // 核心冲突摘要
  plot_summary: string             // 剧情规划（起承转合）
  world_building_summary: string   // 世界观设定
  romance_line_summary: string     // 感情线设计
  style_summary: string            // 风格基调
  checkpoint_summary: string       // 付费卡点设计

  // 结构化数据（供创作阶段使用）
  structured_data: {
    protagonist: { /* 主角详细信息 */ }
    supporting_roles: [/* 配角列表 */]
    checkpoints: [/* 卡点规划 */]
  }
}
```

#### 2.2 跨阶段上下文桥梁

新增 `clarify_context_for_creation` 字段，作为需求澄清到剧本创作的上下文桥梁：

- 保留需求澄清阶段的关键决策记录
- 传递用户的特殊要求和偏好
- 记录需求演变历史

### 3. 剧本创作阶段上下文优化

#### 3.1 剧本创作上下文追踪

新增 `ScriptContext` 结构，实现创作过程的精细化管理：

```typescript
interface ScriptContext {
  // 故事结构定位
  story_phase: '起' | '承' | '转' | '合' | '高潮' | '结局'
  phase_progress: string           // 当前阶段进展描述

  // 人物状态（动态更新）
  character_statuses: {
    [name: string]: {
      current_emotion: string      // 当前情绪
      current_situation: string    // 当前处境
      goal_progress: string        // 目标进展
      relationships: object        // 关系状态
      key_events: string[]         // 经历的关键事件
      secrets_known: string[]      // 已知的秘密
    }
  }

  // 剧情追踪
  completed_events: string[]       // 已完成的关键事件
  pending_hooks: string[]          // 待回收的悬念
  resolved_hooks: string[]         // 已解决的悬念
  foreshadowing: string[]          // 已埋下的伏笔

  // 前情摘要
  recent_summary: string           // 最近3集剧情摘要
  key_revelations: string[]        // 关键揭示/反转
}
```

#### 3.2 故事弧线追踪

新增 `StoryArc` 结构，追踪多线叙事的进展：

```typescript
interface StoryArc {
  arc_type: '主线' | '支线'
  name: string                     // 弧线名称
  status: '进行中' | '已解决' | '待触发'
  trigger_episode: number          // 触发集数
  resolve_episode?: number         // 解决集数
  key_events: Array<{
    episode: number
    event: string
    impact: string
  }>
}
```

### 4. 人物小传丰富性提升

#### 4.1 外观形象维度扩展

人物小传新增完整的外观描述体系：

```typescript
interface CharacterAppearance {
  height: string              // 身高（如：180cm）
  build: string               // 体型（如：身材健硕、纤细苗条）
  hair: string                // 发型发色
  clothing_style: string      // 穿着风格（如：商务精英风、休闲运动风）
  distinctive_features: string // 标志性特征（如：左眉有疤、总戴金丝眼镜）
}
```

#### 4.2 人物字段标准化

新增字段标准化处理，兼容 LLM 返回的各种字段名变体：

```python
def _normalize_character_fields(char: Dict) -> Dict:

        'relationships': 'relationships',
        'relationship': 'relationships',
        '人物关系': 'relationships',
    }
    # 自动转换字段名称为标准格式
```

#### 4.3 人物数量动态调整

根据集数智能调整人物数量：

| 剧情复杂度 | 集数范围 | 主要人物 | 配角数量 |
|-----------|---------|---------|---------|
| 简单剧情 | ≤50集 | 2-3人 | 1-2人 |
| 中等剧情 | 51-80集 | 3-4人 | 2-3人 |
| 复杂剧情 | >80集 | 4-6人 | 3-5人 |

#### 4.4 人物关系图文本化

新增 `relationship_map` 字段，用文字描述人物关系网络，便于理解和导出。

### 5. 界面体验优化

#### 5.1 创作阶段日志精简

**优化前**: 创作阶段显示详细的调试日志面板

**优化后**:
- 移除调试日志按钮和面板
- 简化界面布局，聚焦核心创作进度
- 保留进度条和阶段卡片展示

#### 5.2 导出格式规范化

**新增导出格式配置类**:

- 身高、体型、发型
